Horizon is an operating system that lives entirely on a drive you carry in your pocket and boots on almost any x86-64 laptop with your state intact. The premise is that a computer should not be a specific machine you own but an encrypted object you carry, one that borrows whatever hardware happens to be in front of it. You plug in the drive (a fast NVMe drive in a small USB enclosure, which Horizon calls a Horizon Key), unlock it with a hardware token and a PIN, and a few seconds later you are at your own desktop with your own files and logins, exactly as you left them. When you pull the drive out, the machine remembers nothing. Horizon treats that machine as a disposable Surface that was only ever borrowed glass and silicon.

It runs on the Linux kernel, which is a deliberate choice and not a shortcut. About sixty percent of Linux is device drivers, and rebuilding that is the exact thing that has kept every from-scratch operating system off real laptops for decades, so Horizon takes the kernel as a commodity and spends all of its originality on the layers above it, nearly all of which are written in Rust. The base image is immutable and reproducible, verified at boot with dm-verity, and a single image boots both modern UEFI and older BIOS machines. The first time it meets a laptop it profiles the hardware and caches a tuned configuration, so the next boot on that machine is fast and quiet.

State is handled by the Lifestream, which is probably the part developers will find most interesting. Every change to the system is stored as an encrypted, content-addressed, deduplicated object, so the entire state of the machine at any instant is a single hash in a Merkle DAG. A lot of things that are normally separate and fragile fall out of that one structure for free. Persistence is just the current root. Time travel is checking out an older root, either the whole system or a single file. Updates swap in a new base subtree atomically, rollback keeps the old root bootable, sync replicates the missing objects to another device, and backup is the same operation aimed at a drive you own. For speed it runs on a copy-on-write filesystem and captures into the object store for history and durability, but the thing a person actually feels is that they can rewind their computer like a video.

The second core idea is the Weave, a capability layer that strips ambient authority out of the system. Nothing acts simply because it is running as you. A program, a background service, or the AI can only use capabilities it was explicitly handed, which are scoped and revocable tokens for one file, one network destination, or one device, each of them logged every time it is used. The useful side effect is that application sandboxing, user permissions, and AI safety collapse into one mechanism instead of three. Every program runs inside what Horizon calls a Cell, with no files, network, or devices by default, and has to ask a broker for anything more. A live view called Glass shows every connection and data access as it happens and lets you cut any of them off.

Aura is the on-device AI, and it is built as an ordinary principal inside the Weave rather than something privileged bolted onto the system. It runs locally through llama.cpp on a model sized to the machine, from a three or four billion parameter model on a thin laptop up to larger ones when there is a GPU, with speech handled by whisper.cpp and Piper and semantic search over your own files done with local embeddings. You can type or say something like "find the deck about the cattle project and turn the Q3 numbers into a chart," and instead of clicking blindly around the screen, Aura turns that into a sequence of typed capability calls. It shows you the plan before running it, asks before anything destructive, and everything it does is logged and reversible through the Lifestream, because it can only ever touch what you granted it. If it gets tricked by a malicious instruction buried in a file or an email, it still cannot reach anything it does not hold a capability for. The model is a fine-tuned open one rather than a frontier model trained from nothing, which is the honest and actually reachable version of building your own.

Because the whole point is plugging into machines you do not trust, the security model assumes the host may be hostile. The entire volume is encrypted with LUKS2 and AES-256 and unlocked with a FIDO2 hardware key plus a PIN, so a keylogger on a borrowed laptop is not enough on its own. There is no cloud and no telemetry anywhere in the default system. A mode called Ghost boots completely amnesiac, forces traffic through Tor, and refuses to decrypt your most sensitive data on hardware you do not control. Lose the drive and you can rebuild your identity from secret shares you handed out in advance, with no company ever holding your keys. Horizon is also direct about its limits, which is rarer than it should be: it cannot make a genuinely compromised machine safe while you are sitting at it, because a hardware keylogger or tampered firmware lives underneath anything you can boot.

The same logic runs through the rest of the stack. Rust is used for the capability broker, the state engine, the system services, the compositor, and the shell, with the desktop built on Smithay, iced, and wgpu, following the path System76's COSMIC proved by shipping a from-scratch Rust desktop. The device mesh, called the Constellation, uses QUIC and the Noise protocol to sync between your own machines and, eventually, to let a weak laptop borrow compute from a stronger one you own. The installer is a small Tauri application rather than a web page, since browsers are not permitted to write raw disk images to a drive. The longest-term goal is the most ambitious piece of all: move the trusted core onto seL4, a microkernel that carries a machine-checked mathematical proof of its own correctness, and run Linux inside it as nothing more than a driver layer, so the guarantees become provable rather than merely careful. All of it is open source, partly on principle and partly because a privacy claim that nobody can read the code for is not worth much.
